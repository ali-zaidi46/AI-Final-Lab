I.	INTRODUCTION
Large Language Models (LLMs) are increasingly deployed in real-world systems. However, they are vulnerable to prompt injection, jailbreak attacks, system prompt extraction, sensitive information leakage, and secret exposure. This assignment requires students to design and implement a mini security gateway that protects an LLM-based system from such risks. The focus is on security-oriented system design, quantitative reasoning, and practical engineering implementation.
II.	OBJECTIVES
The objectives of this assignment are:
•	Design a modular LLM security gateway using FAST API/Flask.
•	Detect prompt injection and jailbreak attempts using a scoring mechanism.
•	Detect and anonymize PII using Microsoft Presidio (with customization).
•	Implement configurable policy decisions (Allow / Mask / Block).
•	Evaluate the system quantitatively using structured tables and metrics.
•	Ensure reproducibility through GitHub submission.
III.	SYSTEM REQUIREMENTS
Students must implement the following pipeline:
User Input → Injection Detection → Presidio Analyzer → Policy Decision → Output The implementation must include:
•	Modular code structure.
•	Configurable thresholds.
•	Basic latency measurement.
•	At least three Presidio customizations:
–	Custom recognizer (e.g., phone, API key, internal ID).
–	Context-aware scoring.
–	Composite entity detection.
–	Confidence calibration.
